> **What is Agent Evidence?** [Agent Evidence](https://github.com/joy7758/agent-evidence) is an open-source package for turning agent execution into verifiable evidence bundles. It captures semantic events, stores them in append-only JSONL or SQL backends, and exports signed JSON, CSV, or XML artifacts.

## Installation and setup

Install the package from PyPI:

```bash
pip install agent-evidence==0.2.0 langchain langchain-openai
```

You can also install the latest published version with:

```bash
pip install agent-evidence
```

## Capture LangChain callbacks

Agent Evidence can record LangChain callback events and normalize them into a stable semantic layer such as `chain.start`, `chain.end`, `tool.start`, and `tool.end`.

```python
from agent_evidence import EvidenceRecorder, LocalEvidenceStore, export_json_bundle
from agent_evidence.integrations import EvidenceCallbackHandler
from langchain_core.runnables import RunnableLambda

store = LocalEvidenceStore("data/langchain.evidence.jsonl")
recorder = EvidenceRecorder(store)
handler = EvidenceCallbackHandler(recorder, base_tags=["langchain-demo"])

chain = RunnableLambda(lambda text: text.upper()).with_config({"run_name": "uppercase"})

result = chain.invoke(
    "hello",
    config={
        "callbacks": [handler],
        "metadata": {"session_id": "demo-session"},
        "tags": ["demo"],
    },
)

print(result)

export_json_bundle(
    store.query(source="langchain"),
    "exports/langchain.bundle.json",
)
```

## Capture stream events

If you already consume `astream_events(..., version="v2")`, Agent Evidence can record the emitted event stream without changing your runtime architecture.

```python
import asyncio

from agent_evidence import EvidenceRecorder, LocalEvidenceStore
from agent_evidence.integrations import record_langchain_event
from langchain_core.runnables import RunnableLambda


async def main() -> None:
    store = LocalEvidenceStore("data/langchain-stream.evidence.jsonl")
    recorder = EvidenceRecorder(store)
    chain = RunnableLambda(lambda text: text[::-1]).with_config({"run_name": "reverse"})

    async for event in chain.astream_events("hello", version="v2"):
        record_langchain_event(recorder, event)


asyncio.run(main())
```

## OpenAI Agents exporter

Agent Evidence also includes a companion OpenAI Agents SDK integration built on custom trace processors. This is useful if you want one evidence format across both LangChain and OpenAI Agents workloads.

```python
from agents import trace
from agents.tracing import custom_span

from agent_evidence import EvidenceRecorder, LocalEvidenceStore, export_json_bundle
from agent_evidence.integrations import install_openai_agents_processor

store = LocalEvidenceStore("data/openai-agents.evidence.jsonl")
recorder = EvidenceRecorder(store)
install_openai_agents_processor(recorder, replace=True)

with trace(
    "support-workflow",
    group_id="session-001",
    metadata={"session_id": "session-001"},
):
    with custom_span("collect_context", {"channel": "chat"}):
        pass

export_json_bundle(
    store.query(source="openai_agents"),
    "exports/openai-agents.bundle.json",
)
```

## Export and verify evidence bundles

Agent Evidence supports:

- append-only JSONL storage for local capture
- SQLite and PostgreSQL backends for queryable storage
- JSON, CSV, and XML export
- signed manifests with key rotation and threshold policies

You can export bundles from the CLI as JSON or CSV and attach signed manifests:

```bash
agent-evidence export \
  --store ./data/langchain.evidence.jsonl \
  --format json \
  --output ./exports/langchain.bundle.json

agent-evidence export \
  --store ./data/langchain.evidence.jsonl \
  --format csv \
  --output ./exports/langchain.csv \
  --manifest-output ./exports/langchain.csv.manifest.json \
  --private-key ./keys/manifest-private.pem \
  --key-id evidence-demo
```

Example verification flow:

```python
from agent_evidence import verify_json_bundle

result = verify_json_bundle("exports/langchain.bundle.json")
print(result)
```

## Storage backends

Agent Evidence starts with append-only JSONL storage and also supports SQLite and PostgreSQL backends when you need indexed queries over event type, actor, span identifiers, timestamps, and hash ranges.

```bash
agent-evidence query \
  --store sqlite+pysqlite:///./data/evidence.db \
  --event-type tool.call \
  --source langchain
```

That lets you keep a local audit log during development and move to a queryable backend for larger production traces.
